# 构造函数、原型和原型链

# 构造函数

构造函数本身就是一个函数，与普通函数没有任何区别，不过为了规范一般将其首字母大写。构造函数和普通函数的区别在于，使用 new 生成实例的函数就是构造函数，直接调用的就是普通函数。

constructor 返回创建实例对象时构造函数的引用。此属性的值是对函数本身的引用，而不是一个包含函数名称的字符串。

基本类型的constructor属性值是只读的，引用类型是可以修改的。

# 如何理解原型和原型链？

# 原型

JavaScript 是一种基于原型的语言（prototype-based language），每个对象都拥有一个原型对象，对象以其原型为模板，从原型中继承方法和属性，这些属性和方法定义在对象的构造器函数的prototype属性上，而非对象实例本身。

看一个例子：

function Parent() {
    // ...
}
var child = new Parent()

console.log(Parent.protptype)
console.log(child._proto_)

p._proto_ === Parent.prototype

构造函数Parent上有一个指向原型的指针Parent.prototype，原型上有一个指向构造函数的指针Parent.prototype.constructor。

实例child上有一个child._proto_属性，指向对象的原型。

# 原型链

每个对象都有一个原型对象，通过_proto_指针指向上一个原型，并从中继承方法和属性，同时原型对象也可能拥有原型，这样一层一层，最终指向null。这种关系被称为原型链（ptototype chain）

# 总结

  • 所有的引用类型，都具有对象的特性，即可自由扩展属性
  • 所有的引用类型，都有一个_proto属性（隐式属性）
  • 所有的函数，都具有一个prototype（显式原型），属性值也是一个普通对象。
  • 所有的引用类型，其隐式原型指向其构造函数的显式原型。
  • 当试图得到一个对象的某个属性时，如果这个对象本身没有这个属性，那么会失去它的_proto_中去寻找。
