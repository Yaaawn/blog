# 如何判断数据类型？

先定义一些变量：

var v1 = undefined // Undefined
var v2 = null // Null
var v3 = true // Boolean
var v4 = 1 // Number
var v5 = 's' // String
var v6 = Symbol() // Symbol
var v7 = BigInt(1) // BigInt
var v8 = {} // Object
var v9 = function (){} // Function
var v10 = [] // Array
var v11 = class c {} //
var v12 = new Date() // Date
var v13 = new RegExp('reg') // RegExp
var v14 = Math // 
var v15 = JSON // 
var v16 = new Error('err') //

# 方式一：typeof

先看看结果：

typeof v1 // "undefined"
typeof v2 // "object"
typeof v3 // "boolean"
typeof v4 // "number"
typeof v5 // "string"
typeof v6 // "symbol"
typeof v7 // "bigint"
typeof v8 // "object"
typeof v9 // "function"
typeof v10 // "object"
typeof v11 // "function"
typeof v12 // "object"
typeof v13 // "object"
typeof v14 // "object"
typeof v15 // "object"
typeof v16 // "object"

可以看出，对于基本数据类型，typeof可以很好的判断，而对于引用数据类型，就无法判断出具体的类型。

# 方式二：instanceof

原理：检测 constructor.prototype 是否存在与参数 Object 的原型链上，是则返回 true, 不是则返回 false。

先看看结果：

v8 instanceof Object // true
v9 instanceof Function // true
v10 instanceof Array // true
v11 instanceof Function // true
v12 instanceof Date // true
v13 instanceof RegExp // true
v14 instanceof Object // true
v15 instanceof Object // true
v16 instanceof Error // true

由此可见，instanceof只能用来判断两个对象是否属于实例关系，而不能判断一个对象实例具体属于哪种类型。

# 方式三：constructor

原理：返回对象对应的构造函数。

结果如下：

v1.constructor // Uncaught TypeError: Cannot read property 'constructor' of undefined
v2.constructor // Uncaught TypeError: Cannot read property 'constructor' of null
v3.constructor // ƒ Boolean() { [native code] }
v4.constructor // ƒ Number() { [native code] }
v5.constructor // ƒ String() { [native code] }
v6.constructor // ƒ Symbol() { [native code] }
v7.constructor // ƒ BigInt() { [native code] }
v8.constructor // ƒ Object() { [native code] }
v9.constructor // ƒ Function() { [native code] }
v10.constructor // ƒ Array() { [native code] }
v11.constructor // ƒ Function() { [native code] }
v12.constructor // ƒ Date() { [native code] }
v13.constructor // ƒ RegExp() { [native code] }
v14.constructor // ƒ Object() { [native code] }
v15.constructor // ƒ Object() { [native code] }
v16.constructor // ƒ Error() { [native code] }

可见，null 和 undefined 是无效的对象，没有 constructor，因此无法通过这种方式来判断。

# 方式四：Object.prototype.toString()

原理：默认返回当前对象的 [[Class]] 。这是一个内部属性，其格式为 [object Type] ，是一个字符串，其中 Type 就是对象的类型。

结果：

Object.prototype.toString.call(v1) // "[object Undefined]"
Object.prototype.toString.call(v2) // "[object Null]"
Object.prototype.toString.call(v3) // "[object Boolean]"
Object.prototype.toString.call(v4) // "[object Number]"
Object.prototype.toString.call(v5) // "[object String]"
Object.prototype.toString.call(v6) // "[object Symbol]"
Object.prototype.toString.call(v7) // "[object BigInt]"
Object.prototype.toString.call(v8) // "[object Object]"
Object.prototype.toString.call(v9) // "[object Function]"
Object.prototype.toString.call(v10) // "[object Array]"
Object.prototype.toString.call(v11) // "[object Function]"
Object.prototype.toString.call(v12) // "[object Date]"
Object.prototype.toString.call(v13) // "[object RegExp]"
Object.prototype.toString.call(v14) // "[object Math]"
Object.prototype.toString.call(v15) // "[object JSON]"
Object.prototype.toString.call(v16) // "[object Error]"

可见该方法比较全面，无论是基本类型还是引用类型都能很好的兼容。

# 总结

在实际使用过程中，对于基本数据类型的判断，使用typeof即可，而对于引用数据类型，建议使用Object.prototype.toString()方法。
